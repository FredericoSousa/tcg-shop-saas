## Quando escrever um ADR

Qualquer decisão com impacto arquitetural duradouro: escolha de biblioteca
crítica, modelagem multi-tenant, pattern de concorrência, estratégia de
cache, mudança em contratos de API.

Use numeração sequencial (`0001-...`, `0002-...`) e o template existente
como referência.
